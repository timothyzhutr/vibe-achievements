# Achievement List V1 Draft

This draft defines user-facing achievements for a lightweight macOS app that watches local AI coding conversation history. The achievements should celebrate the user's vibe coding journey: prompting, debugging, tool-hopping, losing and regaining context, accidentally succeeding, rebuilding, shipping, and returning to old projects.

The app itself should stay mostly invisible. Only one or two achievements should be about the tracker; the rest should be about what the user is doing with AI coding tools.

## App / Meta

| # | Achievement | Unlock Definition |
|---:|---|---|
| 1 | **Achievement Unlocked: Unlocking Achievement** | The user unlocks their first achievement. |
| 2 | **Local Legend** | The user keeps the app running locally for 30 days without enabling any cloud sync or account-based features. |

| 5 | **Weekend MVP Energy** | The user describes a project as an MVP, prototype, quick build, side project, or weekend project. |

| 8 | **Side Quest Accepted** | The user starts a new project or feature while another active project already exists. |
| 9 | **Main Quest? Never Heard of Her** | The user works across 10 distinct project histories within one week. |
| 10 | **Keeper of Small Fires** | The user has five active projects with recent AI coding activity. |

## Prompting And Context

| # | Achievement | Unlock Definition |
|---:|---|---|
| 11 | **One More Prompt** | The user continues the same thread for 10 or more user turns. |
| 12 | **Actually, Wait** | The user changes direction mid-thread with language such as "actually", "wait", "never mind", "scratch that", or "instead". |
| 13 | **The Message Had Mass** | A single user or assistant message is unusually long compared with normal conversation messages. |
| 14 | **Confidence High, Context Low** | A long thread continues after the conversation appears to be running low on context, memory, or token budget. |
| 15 | **Context Window Sunset** | The conversation explicitly mentions context limits, compaction, memory loss, token limits, or needing to summarize before continuing. |
| 16 | **Token Budget Lifestyle** | The user discusses tokens, cost, model budget, usage limits, or context management as part of the coding workflow. |
| 17 | **The App Has Opinions** | The assistant pushes back, recommends a different approach, or warns against the user's proposed direction, and the thread continues productively. |
| 18 | **Lore Drop** | The user provides a large amount of background, project history, or prior decision context before asking for a concrete change. |

## Debugging

| # | Achievement | Unlock Definition |
|---:|---|---|
| 19 | **Stack Trace Oracle** | The user shares a stack trace, traceback, compiler error, runtime error, or failed command output in a coding thread. |
| 20 | **The Bug Has A Shape Now** | A vague issue becomes a concrete error, failing test, reproduction step, or suspected root cause within the same thread. |
| 21 | **One More Run** | The same thread includes repeated fix, run, fail, adjust, and retry language. |
| 22 | **Green Bar Acquired** | A test, build, lint, or verification failure is followed later by pass/success language in the same thread. |
| 23 | **Green By Coincidence** | The user or assistant indicates that something passed or started working despite uncertainty about why. |
| 24 | **Understanding Optional** | The conversation includes language such as "I don't know why this works", "not sure why", or "somehow this fixed it". |
| 25 | **We Are So Back** | Failure, brokenness, or error language is followed later by success, fixed, passing, or working language in the same conversation. |
| 26 | **It's So Over** | A thread contains repeated failure, brokenness, or error language without a later success or recovery moment. |
| 27 | **Rubber Duck With A GPU** | The user uses an AI tool to reason through a bug or design problem and reaches a conclusion without asking for a code change. |
| 28 | **The Fix Was Elsewhere** | The thread starts by investigating one area but later resolves or shifts toward a different file, subsystem, dependency, or assumption. |

| 48 | **Again, But Different** | After a project reaches shipping or completion language, the user starts a new project or a major new version. |

## Notes To Review

- Most achievements should be detectable from lightweight local rules and visible conversation metadata.
- Achievement names should feel game-like and meme-aware without being direct references to specific games.
- A few achievements may eventually need tuning so they do not trigger too easily or too rarely.
- The unlock definition describes the user-facing meaning, not the parser implementation.
